task_01.py
Программа принимает список цен и заданный бюджет. Список рассматривается как отсортированный, после чего используются два указателя: один с начала списка, другой с конца. На каждом шаге вычисляется сумма выбранных цен и сравнивается с бюджетом. Если сумма меньше нужной, левый указатель сдвигается вправо, если больше — правый сдвигается влево. При совпадении суммы с бюджетом найденная пара цен возвращается как результат.

task_02.py
Программа принимает строку с названием товара и проверяет её посимвольно с двух сторон. Для этого используются два указателя, которые начинаются с начала и конца строки и постепенно движутся к центру. На каждом шаге символы сравниваются между собой. Если обнаруживается несовпадение, проверка сразу прекращается и выводится сообщение, что строка не является палиндромом. Если все пары символов совпали, строка признаётся палиндромом.

task_03.py
Программа работает со списком товаров, где каждый элемент содержит название и цену. Сначала выполняется сортировка по цене, используя числовое значение стоимости товара. Затем выполняется отдельная сортировка по названию, где сравнение происходит по строковому полю. Отсортированные списки выводятся пользователю для анализа и сравнения.

task_04.py
Программа ищет цену товара по его названию двумя способами. Сначала используется линейный поиск, при котором список товаров просматривается последовательно до нахождения нужного элемента или окончания списка. Затем применяется бинарный поиск, для которого список предварительно сортируется по названию товара, после чего поиск выполняется путём деления диапазона пополам.

Сравнение сложности
Линейный поиск имеет временную сложность O(n), так как в худшем случае требуется проверить все элементы списка. Бинарный поиск имеет сложность O(log n), но работает корректно только на отсортированных данных, поэтому предварительная сортировка является обязательным шагом.
